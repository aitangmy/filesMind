# Security Policy

## Supported Versions

Security fixes are applied to the latest code on the `main` branch.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use GitHub's private vulnerability reporting for this repository when available:

- Repository `Security` tab
- `Report a vulnerability`

If private reporting is not available, open a normal issue with minimal details and request a secure follow-up channel from maintainers.

## Response Process

1. Triage and acknowledge the report.
2. Reproduce and assess impact.
3. Prepare and release a fix.
4. Publish advisory details after remediation when appropriate.
